  • Assertions - Human and machine readable assertions of what service does, leveraging the existing JSON schema, and translating that into business objectives, asserting that a service does what it is supposed to--providing a simple bulleted list explaining what is expected of a service.
  • Scenarios - Providing machine readable definitions that provide the mappings and business rules for orchestrations, scenarios, and other automated processes associated with a service, for use as part of testing, monitoring, security, and other automated functionality.

Always working to aggregate and analyze them, but making sure they are available for use across the API lifecycle.
  • Tags - Using the common tags and taxonomy for operating infrastructure and other resources, allowing for easy discovery, measurement and reporting of what it takes to operate each service.

Conclusion

The production area of the API lifecycle is ongoing as long as any API is still being used in any production applications or system. The stability of production is essential as it feeds back into the discovery, design, and development areas, repeating itself over and over. If things are out of balance, and the essential building blocks are not present in the production phase of the API lifeycle, the required nutrients present in the feedback loop never make it back to the discovery, design, and ultimately development areas of the API lifecycle.
