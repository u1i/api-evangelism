API Outreach

Outreach is required around an API effort, no matter whether purely internal, all about the partners, or pushing for awareness with a public audience. If you aren't reaching out to leadership, stakeholders, across teams, and engaging with API providers and consumers alike, the overall API effort will suffer, always needing more oxygen and awareness amongst those who could help move a platform forward.

API Outreach »Purpose

Defining the purpose behind outreach for API operations will be dependent on the motivations behind doing them in the first pace, and will be something that evolves as a platform matures. Having a clear purpose for API outreach helps regularly remind and guide efforts, keeping them on healthy footing, and having the greatest impact.

API Outreach »Purpose » Build It An They Will Come

It is common for companies, organizations, institutions, and government agencies to launch an API effort with no specifically-stated purpose. They make resources available, build awareness amongst developers, and encourage innovation. While such organizations may not know what the future will hold, they believe that their investment in their API platform in a sensible and pragmatic way will attract developers and the inevitably innovative applications they will bring.

API Outreach »Purpose » A Focus On APIs As A Prduct

Many API providers fully invested in their API efforts, treating their APIs as a product in and of themselves. They develop formal programs around their API operations, treat API consumers as end customers, and consistently ensure their programs have the resources they need. In short, they treat API operations like a business and run them as efficiently as possible, even if commercial sales is not the end goal.

API Outreach »Purpose » A Focus On APIs As An Enabler

Many API providers emphasize ensuring the availability of APIs as a means to support an existing web or mobile application; in a sense, the API is relegated to a secondary role relative to the primary application’s existence. APIs for these kinds of organizations serve to drive traffic, adoption, and integration when it comes to a larger vision, but the APIs themselves are simply enablers for the overall platform.

API Outreach »Purpose » A Focus On The Developer

Beyond the API as a product — and the web and mobile apps they power — API efforts tend to focus on the developer. Development-focused APIs emphasize what developers will bring to the table when they have the resources and support they need, and are thus central to the investment and engagement necessary for successful outreach/development.

API Outreach »Purpose » Leverage Network Effect

Some API providers we interviewed expressed an interest in joining a wider network of collaborative open data and API efforts. They felt that working in concert with others, by bringing together many different federal, state, or municipal agencies, would benefit the platforms. Further, they felt this would allow the API initiatives to be led by either policy or private interests.

API Outreach »Purpose » Increased Efficiences

Overall, the most positive motivation for API development expressed by existing providers is to streamline API on-boarding and integration. Further development would help internal, partner, and 3rd party public developers be more successful in putting API resources to work in their web, mobile, and device applications.

API Outreach »Purpose » Attract Interesting Organizations, Projects, and People

API platforms are often aimed at attracting the attention of interesting/progressive companies, universities, institutions, and other entities. These external entities can often put API resources to use in new and innovative ways, and in doing so, they can bring attention and potential partnerships to the platform.

API Outreach »Scope

Defining what the scope of API outreach will look like based upon the resources available, and the goals established for conducting outreach. Setting realistic expectations for what is possible, and help ensure the outreach effort is as efficient and effective as possible.

API Outreach »Scope » Starting Small

A common theme across API operations is the importance of starting small and building a stable foundation before attempting larger infrastructure development. Beginning with a basic, yet valuable set of API resources, fleshing out the entire lifecycle, and processing around what is necessary to be successful before scaling and increasing the scope of API operations was routinely described as a critical part of any API scope.

API Outreach »Scope » Investing When It Makes Sense

A lack of resources is a common element of slow growth and unrealized API operations. Bringing significant levels of investment to projects while simultaneously applying appropriate human and technological capital to API operations were universally mentioned by our interviewees as critical to seeing desired results.

API Outreach »Scope » Working With What You Have

Almost all the API programs are forced to work in resource-starved environments. They wished to do more and to be more, but lacked the time, human investment, and technical resources to make it happen. This forced the organizations to work with what they had, making the most of their limited opportunities while hoping for eventual greater contributions.

API Outreach »Scope » Everyone Pitches In

While a dedicated team is a requirement, an API’s development should always invite individuals from across an organization to join in the process. Ideally, API operations are a group effort led by a central team. It is important to encourage individual API teams to feel a sense of ownership over their API, just as it is important to encourage business users to participate in development conversations.

API Outreach »Scope » A Focus On Constant Improvement

Most API programs will express that they always want to be doing more with their programs. They want to formalize their approach and increase their resource and labor investment in the areas in which they have seen success. Ultimately, their target scope was to focus not on just meeting expectations, but on moving towards excellence and mastery of what it takes to scale their API efforts.

API Outreach »Scope » Dedicated Teams

Even if it is a small group, always work to dedicate a team to API operations. While APIs will ultimately be an organization-wide effort, it will take a dedicated team to truly lead the API to success. Without centralized, dedicated leadership, APIs will never attain true relevance within an organization, leaving to be a side project that rarely delivers as expected.

API Outreach »Structure

Establishing a structure for API outreach, helping define what operations looks like, who is involved, and how it will be conducted on a weekly and monthly basis. Helping formalize how outreach occurs, keeping it moving forward, scaling, and providing as much value as it possibly can.

API Outreach »Structure » Dedicated Evangelist(s)

Make sure there is a dedicated evangelist or group of evangelists to help talk-up and support the platform internally, with partners, publicly, or based upon any other scope determined sensible.

API Outreach »Structure » Include Other Teams

Do not just invest in an isolated API team; make sure other teams across the organization are included in the API conversation.

API Outreach »Structure » Use a CRM

Put a CRM to use when it comes to tracking and automating outreach efforts for the platform. Do not reinvent the wheel; leverage an existing service to track all of the details that can be automated.

API Outreach »Structure » Develop In-Person Presence

Make sure to obtain human resources that can be sent to meetups, conferences, and other in-person events so that the organization can expanding and strengthening the presence of the platform.

API Outreach »Structure » Involve Leadership

Regularly invest time to report platform results and progress to leadership, making sure they understand the overall health and activity of the platform.

API Outreach »Structure » Dedicated Team

Allocate a dedicated team to the API program and platform to represent its needs and push for its continued advancement.

API Outreach »Structure » Business Involvement

Make sure to involve business groups in the conversation, bringing them in as stakeholders to give their thoughts on the organization’s API program.

API Outreach »Structure » Marketing Involvement

Make sure marketing teams play an influential role in amplifying the message coming from a platform. This is especially important to ensure that a platform does not just speak to developers, but to consumers and users as well.

API Outreach »Structure » Sales Involvement

Ensure that a platform has the sales resources to actually close deals when necessary. This ensures that a platform has continuing end-user participation and the resources it needs to function. Remember, though, that sales is not always about selling commercial solutions.

API Outreach »Structure » Dedicated Program

Establish a formal program for overseeing API operations across an organization to make sure it gets the attention it needs.

API Outreach »Defining Success

Success of API outreach will depend on a lot of motivating factors behind the platform, and the personalities involved with it's execution. Having a coherent definition of what success means when it comes to API outreach is essential to ever realizing the vision of the platform, especially as it will become a moving target.

API Outreach »Defining Success » Building Awareness

API success revolves around building awareness that an API exists, as well as awareness of the value of the API resources that are being made available. Awareness is not simply a consumer side consideration, though; providers, too, must possess an awareness of the value of their resources in relation to both other API developers and consumers alike.

API Outreach »Defining Success » Attracting New Users

Bringing attention to an API and attracting new users is one of the most common ways of measuring the success of API operations. While new users won’t always immediately become active users, their interest and involvement will bring attention to and awareness of what the API platform can deliver. Attracting new users is one of the easiest and most accessible ways to measure the success of any API, according to our interviews, but importantly, none of the platform providers we spoke to recommended that an organization

API Outreach »Defining Success » Incentivizing Active Users

While attracting new users is easy, producing active users is much harder. The easier it is to onboard with an API and make the first series of API calls, the greater the likelihood that API consumers will integrate the platform into their own resources and work, which is a critical metric for any API provider.

API Outreach »Defining Success » New Applications

Applications and development are the cornerstones that incentivize API providers to invest in APIs, and across the board, our interviewees cited application integration and involvement as a prime candidate for determining an API’s success. This could be quantified both in terms of new applications relying on the API platform, as well as active application processes that integrate with the API’s platform. In either case, measuring usage was considered an excellent means to justify the existence and growth of an API platform.

API Outreach »Defining Success » New Partnerships

Getting the attention of companies, organizations, institutions, and other government agencies is an important part of the the API journey. In particular, developing an awareness of and encouraging the usage and adoption of APIs among groups already leveraging the technology is an important metric by which success can be determined.

API Outreach »Defining Success » New Services

An additional area that was used to define success was the number of new API resources added to a platform. If an organization is currently in the development phase and deploying APIs into production, it is likely that a platform already has a handle on what it takes to successfully deliver APIs throughout their lifecycle. Making new APIs a great way to understand the velocity of any platform, and how well it is ultimately doing.

API Outreach »Defining Success » Serving End-Users

Of course, API providers articulated the importance of serving end-users. Besides serving an organization’s mission, the true purpose of an API is to satisfy an end-user, and so measuring success based upon how much value is created and delivered to these users and customers, and even the public at large, can directly verify that an API is living up to its billing.

API Outreach »Defining Success » Knowing Your Resources

Understand the resources you are making available via an API. Ensure that they are well defined and made accessible in ways that consider security, privacy, and the interests of all stakeholders. Do not just open up APIs for the sake of delivering APIs — make sure the resources are well defined, designed, and serve a purpose.

API Outreach »Defining Success » New Revenue

For many API providers API success will always be measured in the form of new revenue. Either increased from existing services, or increased from new services, either way measuring all outcomes based upon revenue.

API Outreach »Evangelism

Work hard to reach out to people, helping bring awareness to APIs in general, but also around specific aspects of the platform, getting them pumped and more engaged with what is happening--ultimately turning anyone who is willing into an evangelist for the cause.

API Outreach »Evangelism » New Engagement

Dedicating time to reaching out to new users, and potential consumers, partners, and stakeholders who have not be engaged with yet. Expanding the reach of the API effort, converting more believers, and getting them involved with the effort.

API Outreach »Evangelism » Events

Spending time a meetups, hackathons, conferences, and other relevant events externally and internally within the enterprise. Ensuring there is a regular evangelism presence at the most important events.

API Outreach »Evangelism » Existing Consumers

Investing time in reaching out to existing users, understanding what they are doing, strengthening relationships, deepening engagements, and making sure past work continues to pay off when it comes to outreach and evangelism.

API Outreach »Evangelism » Leadership

Making sure leadership is always being evangelized to, pushed to be more involved and aware of what is happening with API operations. Ensuring the API operations is never in a blind spot when it comes to decision making at the highest levels.

API Outreach »Communication

Having a communication strategy is key to API outreach, being organized about the message you put out, and ensuring that all stakeholders stay up to date on what is happening, and how a platform is making an impact on their world.

API Outreach »Communication » Blogging

Providing regular stories on an internal, partner, or public blog regarding what is happening with a platform, sharing ongoing insight into what is happening behind the scenes, and across teams.

API Outreach »Communication » Social Media

Using public or private social media channels to engage with stakeholders, partners, customers, and the public when it comes to what is happening with a platform, providing visibility into what is going on.

API Outreach »Support

Being there to support anyone engaging with a platform is a great way to reach out, stay engaged, and let them know that they are important to a platform's operation. Providing helpful, timely, and responsive support is the best form of outreach for any API platform.

API Outreach »Support » Self-Service

Providing self-service support channels that help re-enforce the relationship with users, letting them know there is someone home that cares about them, providing them with up to date, relevant resources to get help.

API Outreach »Support » Direct Support

Provide several direct support channels where developers can directly engage with API owners, removing too many degrees of separation between provider and consumer, strengthening the relationship.

API Outreach »Support » Paid Support

Providing paid tiers where API consumers can actually pay to get the attention they need, establishing a professional relationship between provider and consumer, letting them know they can always get the help they need.

API Outreach »Road Map

Any platform is perpetually moving forward, heading towards some destination far off on the horizon, and sharing the road map for where a platform is going is essential API outreach being meaningful and effective. Sharing with all stakeholders, what the future will hold keeps everyone engaged.

API Outreach »Road Map » Engaged Consumers

Reaching out to engaged users regarding a private road map, and giving them an early look at the road map, letting them provide feedback, and giving them a head start when it comes to the platform's direction.

API Outreach »Road Map » Communication

Provide a regular drumbeat of communication surrounding releases, keeping consumers in tune, and educating them about what is happening with the platform, and the direction things are headed.

API Outreach »Resources

Making sure there are plenty of up to date, relevant, and valuable resources for API providers and consumers to take advantage of is essential to outreach efforts, providing asynchronous materials that can help extend the reach and capacity of a team without adding more team members.

API Outreach »Resources » White Papers

Providing a regular stream of white papers produced by the platform, by partners or relevant ones from the industry that help engage with consumers, keeping them more engaged and aware.

API Outreach »Resources » Use Cases

Publishing real world use cases of what is possible with a platform, and services being offered. Providing detailed introduction to relevant use cases within industries, or providing solutions to real problems.

API Outreach »Resources » Case Studies

Publishing case studies of how other teams, partners, and public consumers are putting API resources to work, demonstrating real world relevance, and helping API consumers realize wht is possible on a regular basis.

API Outreach »Training

Making sure there is a regular stream of training opportunities for API providers, consumers, and other stakeholders to tap into when it comes to learning what they need about a platform, its services, and the industry that they serve.

API Outreach »Training » Workshops

Providing regular series of workshops that provides deeper dives into topics surrounding API operators, providing face to face time with knowledgable API operators, who can answer their questions and move forward their understanding of what is possible.

API Outreach »Training » Webinars

Providing regular webinar series that help consumers with common topics and things they face when working with API resources. Providing a virtual presence they can tap into to learn about what is going on, and reminding them that someone is out there.

API Outreach »Virtualization

Providing a sandbox for engaging with providers, consumers, and other stakeholders. Providing a safe place to understand what is possible, engage with the community, and evolve the relationship between provider and consumer, while also benefitting the platform.

API Outreach »Virtualization » Dedicated Sandboxes

It is often times much easier to engage with developers and help them along with an integration in a sandbox, then it is in a production API environment. Allowing developers to safely learn about what an API does, before they move to far forward.

API Outreach »Virtualization » Virtualize APIs

Virtualize APIs: Provide virtualized instance of APIs and a sandbox catalog of API resources that reflect production resources help quickly build and strengthen relationships with users who may not have felt comfortable working in a production environment.

API Outreach »Virtualization » Virtualize Data

Provide virtualized and synthesized data along with APIs in order to create as realistic an experience as possible, helping developers quickly understand what is possible, and develop their integration capabilities regarding what is possible without any negative consequences.

API Outreach »Virtualization » Virtualized Containers

Consider the availability of virtualized instances of an API as containers, allowing API sandboxes to be created locally, in the cloud, and anywhere containers run. Embedding virtualized instances of APIs within developers work environment for continued exposure.

API Outreach »Virtualization » Certification

Provide certification for both developers and applications. Establishing a minimum bar for what is expected of applications before they can move into production helps developers understand what it takes to move an application from sandbox to distribution, which ensures a high-quality application experience at scale, and a deeper more sustained commitment.

API Outreach »Management

Leveraging the API management layer when it comes to outreach, making sure you are able to engage with all providers and consumers, and taking advantage of the opportunities that exist around platform operations. Ensuring that your platform providers and consumers are doing what they should be, and being reached out to regarding what their platform usage looks like.

API Outreach »Management » Portal

Making sure the API portal is always up to date, fresh, and providing what visitors are needing. Making sure API providers and consumers are engaged across the enterprise, with partners, and the public when it is relevant.

API Outreach »Management » Authentication

Require authentication for all API resources, ensuring that there is a name and email associated with every consumer. You can't reach out to consumers if you do not know who they are. Providing the link for other forms of engagement on a regular basis.

API Outreach »Management » Dashboard

Providing a provider and / or consumer dashboard, showing their service usage, and providing other resources, information, communication, support, and visualizations that help keep them engaged and maximizing engagement.

API Outreach »Event-Driven

Taking advantage of event-driven opportunities and investment to stay engaged with API providers and consumers, delivering real time streams of data, pushes and notifications of what is changing. Elevating platform activity to be more about outreach and engagement in real time, rather than waiting for consumers to come to you with a request.

API Outreach »Event-Driven » Webhooks

Webhook subscriptions and email notifications can keep a platform on the radar of individual developer, companies, and other entities. Providing an avenue for keeping a steady heartbeat of outreach ocurring.

API Outreach »Event-Driven » Streams

Ensuring there are regular subscriptions, topics, and streams that can be subscribed to, ensuring consumers are engaged in a real-time way if they desire, establishing a more sustained form of outreach.

API Outreach »Observability

Leverage the existing outputs that exist across the API lifecycle as a way of reaching out to consumers in real time, providing a heartbeat of the platform that assures them all is well, and that the platform is doing what is expected. Turning the heartbeat of operations into an ongoing stream of outreach to the platform providers and consumers who have access.

API Outreach »Observability » Management

Providing observability into platform consumption for all API providers and consumers, delivering a steady drumbeat of information and outreach occurring regarding what is happening across a platform.

API Outreach »Observability » Monitoring

Offering a status dashboard, and regular notifications regarding platform availability and health is a great way to maintained sustained outreach and presence within providers and consumers minds.

API Outreach »Observability » Testing

Providing access to testing services that allow providers and consumers to better test the API infrastructure they are responsible for or dependent upon. Scheduling, saving, archiving, and responding to test-driven events that are occurring.

API Outreach »Observability » Performance

Staying engaged with API providers and consumers by providing them with performance updates, notifications, and other meta data regarding the APIs they are responsible for, or dependent on.

API Outreach »Observability » Security

Using platform security s a way of staying engaged with providers and consumers, providing them with regular information regarding how the platform is being secured, and how both API providers and consumers can better stay secure.

API Outreach »Observability » Road Map

Sharing future plans for the platform with all stakeholders, providers, and consumers, giving them a heads up regarding what the future holds, while also potentially giving them a voice in helping steer the ship.

API Outreach »Sustaining Adoption

Reaching out to platform stakeholders to help increase engagement, and ensure they will be sticking around as active users for the long term. Strengthening the relationships that already exist, and pushing them to be in closer alignment with platform and outreach goals.

API Outreach »Sustaining Adoption » Management

Almost all API providers we talked to articulated that having a proper strategy for API management, as well as an investment in services and tooling, was essential to onboarding new consumers. Additionally, this kind of investment facilitates an understanding of how to engage with and incentivize the usage of API resources by existing users. Without the ability to authenticate, define access tiers, quantify application usage, log all activity, and report upon usage and activity across dimensions, it is extremely difficult to scale platform adoption.

API Outreach »Sustaining Adoption » Feedback Loops

Having feedback loops in place are essential to driving and sustaining the adoption of APIs. Without one or more channels for consumers to provide feedback, as well as responsive and attentive API providers who analyze how the feedback can fit into the overall API plan, API operations will never quite rise to the occasion.

API Outreach »Sustaining Adoption » Proactive

Make sure you are proactive in operating your API platform by constantly marketing your work to developers (remember, continually attracting new attention is vital). At the same time, work to provide existing developers with what they will need based upon common practices across the API sector. Investing in developers by giving them resources they will need before they have to ask for it makes an API’s community feel alive and healthy.

API Outreach »Sustaining Adoption » Reactive

While proactivity is important, an API team must also be able to react to any questions, feedback, and concerns submitted by API consumers and other stakeholders. Ensuring people do not have to wait very long for an answer to their question makes consumers, developers, and stakeholders alike feel like they are considered a relevant and important part of the API community.

API Outreach »Sustaining Adoption » Content

Content is critical. Invest in blog posts, samples, tutorials, case studies, and anything else you think will assist your consumers in their journey. We heard over and over how important a regular stream of content is for attracting new developers, keeping active ones engaged, and putting API resources to work.

API Outreach »Sustaining Adoption » Code

Providing samples, SDKs, start solutions, and other code elements is vital to making sure developers understand through demonstration how to integrate with APIs in a variety of programming languages. Code will increase their dependency on the services being offered, and support them in expanding that surface area.

API Outreach »Sustaining Adoption » Event-Driven Infrastructure

Sustained adoption will be defined by the event-driven possibilities, which can also be used to incentivize deeper engagement and workflow with an API. Event-driven tools like webhooks, streaming API, and delivery via topics, schedule, and event types help augment existing APIs and make them more usable by API consumers at scale. They facilitate a meaningful application experience for end-users, allowing them to stay tuned to specific topics. This in turn fine-tunes the experience for developers,which further drives adoption, ultimately establishing more loyal consumers at the API integration and application user levels.

API Outreach »Landscape Analysis

Ensuring that API outreach is informed, and targeting everywhere it should be, through the development of an awareness of the landscape that exists internally, with partners, competitors, and any other slice of space in which services will be delivered.

API Outreach »Landscape Analysis » Industry Level

Understanding the industries where services are being delivered, and having the landscape mapped, developing an understanding of where services fit into the bigger picture.

API Outreach »Landscape Analysis » Enterprise

Mapping out the organizational landscape that exists across the enterprise, defining groups, teams, programs, projects, and other logical separators that make up the playing field, while also establishing a deep understanding of who all the players are, or will be.

API Outreach »Landscape Analysis » Competitive

Having a clear map of the competitive landscape, mapping out all competing interests, with as much due diligence regarding how they compare. Understanding what it takes to out maneuver the competition, and be more agile, nimble, and innovating than anyone else on the field.

API Outreach »Landscape Analysis » Domain

Mapping out the domains that exist, ensuring the platform is being operated with the greatest awareness of other standards, specifications, vocabulary, regulations, and tooling that exists to inform operations regarding specific domains being operated within.

API Outreach »Partners

Giving extra special attention when it comes to reaching out to the most trusted and valuable partners, making sure they have priority when it comes to outreach efforts and resources. Investing more heavily in partners so that they can help move API outreach efforts forward by amplifying and extending the reach of what is going on.

API Outreach »Partners » Communication

Providing regular communication that targets partners, ensuring there is a steady flow of information headed their way to keep them informed of partner opportunities, and what is going on with a platform.

API Outreach »Partners » Showcase

Regularly showcase what partners are doing, and especially what they are doing around platform services, helping amplify the partnership, and showcase what is possible with services.

API Outreach »Partners » Marketing Activities

Offer up a regular buffet of marketing activities that are available to partners, giving them endless opportunities to be further engaged, and part of the platform story.

API Outreach »Partners » Support

Provide partner level tiers of support that focus on the needs of the most high value partners, and giving what they need to be successful, and further the platfor objectives.

API Outreach »Reporting

Leveraging operational level reporting as a part of the outreach efforts. Leveraging operational data from across the platform and turning it into relevant signals, visualizations, and other educational forms for platform stakeholders to consume.

API Outreach »Reporting » Stakeholders

Delivering regular, up to date, informative reports on what is happening with a platform to all stakeholders, helping them understand the value being generated, and keeping them engaged with what is going on.

API Outreach »Reporting » Consumers

Regular reports in the inbox, on the dashboard, and via notifications helps keep consumers engaged and aware, providing them with relevant data regarding how they can optimize their usage of the platform.

API Outreach »Reporting » Leadership

Provide a steady stream of executive reporting to leadership, ensuring they have what they need to stay informed, and providing a regular and steady stream to perpetually reach out and keep them engaged.

API Outreach »Conclusion

Outreach will depend on the motivations behind a platform, the resources available, and the people who roll up their sleeves and get their hands dirty doing the work. API outreach is something that is shared by all, and lead by a passionate few, helping perpetually spread awareness around an API platform, and the value it can deliver when put to work by both API providers and consumers, when it comes to driving the next generation of applications.
