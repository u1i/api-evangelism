API Governance

API governance is the ultimate goal of wanting to establish a formal strategy for delivering APIs consistently at enterprise scale. It is what is needed to quantify, measure, and ultimate steer the enterprise API ship, and guide teams forward when it comes to delivering APIs at scale. Governance will mean many different things to different organizations, and team members, but there will always be some essential building blocks we should be looking at to understand where we are succeeding or failing with our API efforts.

API Governance »Structure

Working to understand the scope and structure for governance operations, establish the right approach to understanding API operations across an organization, and be effective when working with many different teams, operating at different levels of maturity.

API Governance »Structure » Resources

Defining the resources that are available to realize governance. Being realistic about what is possible, and what it will take to actually make change across the organization.

API Governance »Structure » Distribution

How distributed or centralized will API governance be. Reflecting the organization(s) it represents, but appropriate for moving forward the conversation as desired.

API Governance »Structure » Team

Making sure there is someone present from every team, representing the governance effort, requiring that all teams have a seat at the table, providing feedback on the overall governance, but also acting as coach and leader within each team when it comes to governance.

API Governance »Discovery

Ensuring that all APis are discoverable and able to be measured as part of overall governance. If it isn\'t known about it can\'t be monitored, tested, secured, and analyzed for the value it generates across an organization.

API Governance »Discovery » Organization

Have a clear map, as well as understanding of the organization, and the breakdown of API stakeholders across those teams. Possessing a machine readable way that is integrated with existing directory solutions, as well as overlays and takes advantage of existing services and software in use.

Having a clear definition of the domains involved with enterprise operations. This is a tricky world to define because it is spread across Internet domain, as well as expertise domains, which may or may not overlap. Regardless, working to have a clear map of domains across the enterprise, and an understanding of how API infrastructure is operated across them is essential.

API Governance »In Conclusion

The purpose of this workshop is to help introduce one possible approach to quantifying the API lifecycle as it exists across the enterprise, providing a map of the enterprise landscape, so that it can be quantified, articulated, and evolved as a coherent, organization-wide strategy. API governance represents the end goal for many enterprise groups, but in reality, each area of the API lifecycle showcased here will continue to be just as important, providing the framework to navigate the landscape, as well as the artifacts for measuring, quantifying, and adjusting the enterprise digital transformation for years to come.
